docker,kubernetes 이용하여 aws 배포하기

이번에 이직한 회사에서 배포를 위해 docker 랑 kubernetes 를 사용하고 있다
이때까지 devops 엔지니어분이 jenkins 를 세팅해주시면
node 버전정도만 가볍게 수정해서 배포를 했기떄문에 docker 랑 kubernetes 에 대해 잘 모른다
그래서 docker kubernetes 를 사용해서 aws 에 배포하는작업까지 해보려한다

우선 docker 에 대해서 알아보자

Docker

도커(Docker)는 컨테이너 기반 가상화 플랫폼으로
애플리케이션을 개발, 배포, 실행하기 위한 환경을 제공한다.

Docker는 애플리케이션 개발 배포 실행 하기 위한 환경을 제공하는 가상화 플랫폼이라는것 까지는 이해가 됐다 그런데 컨테이너는 어떤걸 말하는걸까

컨테이너

컨테이너는 애플리케이션이 한 컴퓨팅 환경에서 다른 컴퓨팅 환경으로 빠르고 안정적으로 실행되도록 코드와 모든 종속성을 패키징하는 소프트웨어의 표준 단위입니다.

한마디로 Docker란 여러 컴퓨팅 환경에서도 빠르고 안정적으로 실행할수 있는 환경을 제공하는 기술 이라고 이해를 했다

그러면 kubernetes 는 어떤것인지 알아보자

kubernetes

K8s라고도 하는 Kubernetes는 컨테이너화된 애플리케이션의 배포, 확장 및 관리를 자동화하기 위한 오픈 소스 시스템입니다.

즉 Docker 를 사용하여 내 프로젝트를 컨테이너화 한 뒤
kubernetes 를 이용하여 컨테이너 된 프로젝트를 자동으로 배포 확장 및 관리 를 하는 기술 이라고 이해를 했다

aws

이제 aws 에 대해 알아보려 한다 aws 는 아마존에서 제공하는 웹 서비스(Amazon Web Services)이다
aws 에서는 정말 다양한 서비스를 지원하고 있다

이전 회사에 다닐때는 s3, colud front 등 다양한 기술을 사용했던것으로 기억하는데 자세히 관심을 가지지 않았었던게 아쉽다.

일단 내가 어떤 서비스를 이용해야하는지 부터 확인해보겠다
aws 한국블로그 에 따르면
Lightsail, Elastic Beanstalk 둘중에 하나를 선택하는게 좋아보인다.

현재 적은 트래픽을 가진 단일 웹 페이지를 배포할 예정이므로
Elastic Beanstalk 을 사용하는게 조금더 적합해 보이지만 지금 다니는 회사가 lightsail 을 stage 서버용으로 사용하고 있으니 이에대해 공부도 할겸 lightsail 을 이용을해보고 추후 여유가 생길때 나머지 서비스에 대해서도 알아보도록 하겠다

그러면 이제 lightsail 에 대해 알아보자

lightsail

Amazon Lightsail은 가상 프라이빗 서버(VPS) 공급자로, 클라우드에서의 애플리케이션 구축 및 호스팅 솔루션이 필요한 개발자, 소규모 비즈니스, 학생 및 다른 사용자가 가장 손쉽게 AWS를 시작할 수 있는 방법입니다. Lightsail은 개발자에게 클라우드에서 웹사이트와 웹 애플리케이션을 배포하고 관리할 수 있는 컴퓨팅, 스토리지 및 네트워킹 용량 및 기능을 제공합니다. Lightsail에는 프로젝트를 빠르게 시작하는 데 필요한 모든 것(가상 머신, 컨테이너, 데이터베이스, CDN, 로드 밸런서, DNS 관리 등)이 포함되어 있으며, 이러한 서비스를 저렴하고 예측 가능한 월간 요금으로 사용할 수 있습니다.

lightsail 은 vps 공급자로 클라우드에서 웹사이트와 웹 애플리케이션을 배포하고 관리할 수 있는 컴퓨팅, 스토리지 및 네트워킹 용량 및 기능을 제공한다 로 요약할수 있을거 같다

이제 대충 어떤기술 사용할지 정했으니 다음 포스트에 어떻게 하는지 알아보도록 하겠다.

D3

D3 를 공부하려고 한다
workbox, D3, 테스트 코드 3개를 공부해 보려 하는데
먼저 D3 부터 시작하기로 했다
D3란 Data-Driven Documents 의 약자로 D가 3개 들어가서 D3 라고 하는거같다 직역을 하면 정보-중심 문서 라는 뜻인거같다
일단 d3js.org 에 접속해 보았다, 접속하자마자 최 상단에 d3 가 뭘 하는것인지 쓰여있는데

D3.js is a JavaScript library for manipulating documents based on data. D3 helps you bring data to life using HTML, SVG, and CSS. D3’s emphasis on web standards gives you the full capabilities of modern browsers without tying yourself to a proprietary framework, combining powerful visualization components and a data-driven approach to DOM manipulation.

대충 해석해 보자면
D3 는 데이터 기반으로 문서를 조작하는 자바스크립트 라이브러리 이다
HTML,SVG,CSS 에 생명을 불어 넣는데 도움이 된다고 한다
웹표준을 준수하여 프레임 워크에 묶이지 않고도 최신 브라우저의 기능을 제공한다
영어실력이 좋지 못해서 정확한 해석은 되지 않지만 대충 이런 뜻인거 같다
이후 쭉 읽어보면 D3 에대한 상세한 설명 후 아마도 핵심 기능으로 추정되는
5가지 기능에 대한 설명이 있다

  1. selections
  2. Dynamic Properties
  3. Enter and Exit
  4. Transformation, not Representation
  5. Transitions

따라서 위 5가지 순서대로 공부후 나중에 예제를 몇개 만들어 보려고 한다

git hub 에 SSH key 등록하기

git hub 에 ssh key 를 등록하는 방법에 대해서 알아보도록 하자
mac os 기준으로 SSH key 를 등록할것이니 대부분 비슷하겠지만
mac 이 아니라면 명령어가 조금 다를수도 있다

SSH key 란

서버에 접속 할 때 비밀 번호 대신
사용하는 기능이다
비밀번호 보다 높은 수준의 보안을 필요로 하거나 로그인 없이 자동으로 서버에 접속 할 경우 사용한다
SSH key 는 비대칭 암호방식을 사용한다
public key 와 private key 2가지를 생성한다
public key 는 평문을 암호화 private key 는 암호문을 다시 평문으로 변환한다그래서 public key 를 github 에 업로드 하고
private key 는 로컬에만 두고
대조를 해서 사용자 를 인증을 하는 방식이다
이제 SSH key 를 등록하도록 해보겠다

목차

  1. SSH key 확인
  2. SSH keygen</
  3. SSH key copy
  4. gih hub 등록

1.SSH key 확인

우선 터미널에 들어가 위 명령어를 순서대로 입력한다
만약 SSH key 를 생성한 적이 있으면 이때 공개키가 있는것을 확인할수 있다
공개키가 있으면 2번 단계를 건너뛰고 3번으로 가면 된다.

2. SSH keygen

위 명령어를 입력하면 ssh key 가 생성이 될것이다
-t 는 타입을 지정해주는 옵션이다
-C 는 코맨트 를 입력해주는 옵션이다
-b 는 비트수를 입력하는 옵션이다
그래서 rsa 타입으로 2048 비트 인 SSH key 를 생성해준다는 뜻이다
2048비트와 rsa 타입은 기본 값이므로

으로만 입력해도 github에 등록하는데는 문제가 없다

2. 3. SSH key copy

이제 SSH key 를 생성 했으니 복사 하도록 하겠다

명령어를 입력하면
공개키가 터미널에 나올것이다 그부분을 복사하면 된다

명령어를 통해 바로 클립보드로 복사할수도 있다

4. git hub 등록

ssh public key를 클립보드에 복사했다면
github 에 접속후 우측상단에 프로필을 클릭한다음 setting 에 들어간다

그후SSH and GPG keys 에 들어간뒤 new SSH key 를 클릭한다

Key 부분에 복사한 public key 를 등록하면 SSH key 등록이 된다
