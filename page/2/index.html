git hub 에 SSH key 등록하기

git hub 에 ssh key 를 등록하는 방법에 대해서 알아보도록 하자
mac os 기준으로 SSH key 를 등록할것이니 대부분 비슷하겠지만
mac 이 아니라면 명령어가 조금 다를수도 있다

SSH key 란

서버에 접속 할 때 비밀 번호 대신
사용하는 기능이다
비밀번호 보다 높은 수준의 보안을 필요로 하거나 로그인 없이 자동으로 서버에 접속 할 경우 사용한다
SSH key 는 비대칭 암호방식을 사용한다
public key 와 private key 2가지를 생성한다
public key 는 평문을 암호화 private key 는 암호문을 다시 평문으로 변환한다그래서 public key 를 github 에 업로드 하고
private key 는 로컬에만 두고
대조를 해서 사용자 를 인증을 하는 방식이다
이제 SSH key 를 등록하도록 해보겠다

목차

  1. SSH key 확인
  2. SSH keygen</
  3. SSH key copy
  4. gih hub 등록

1.SSH key 확인

우선 터미널에 들어가 위 명령어를 순서대로 입력한다
만약 SSH key 를 생성한 적이 있으면 이때 공개키가 있는것을 확인할수 있다
공개키가 있으면 2번 단계를 건너뛰고 3번으로 가면 된다.

2. SSH keygen

위 명령어를 입력하면 ssh key 가 생성이 될것이다
-t 는 타입을 지정해주는 옵션이다
-C 는 코맨트 를 입력해주는 옵션이다
-b 는 비트수를 입력하는 옵션이다
그래서 rsa 타입으로 2048 비트 인 SSH key 를 생성해준다는 뜻이다
2048비트와 rsa 타입은 기본 값이므로

으로만 입력해도 github에 등록하는데는 문제가 없다

2. 3. SSH key copy

이제 SSH key 를 생성 했으니 복사 하도록 하겠다

명령어를 입력하면
공개키가 터미널에 나올것이다 그부분을 복사하면 된다

명령어를 통해 바로 클립보드로 복사할수도 있다

4. git hub 등록

ssh public key를 클립보드에 복사했다면
github 에 접속후 우측상단에 프로필을 클릭한다음 setting 에 들어간다

그후SSH and GPG keys 에 들어간뒤 new SSH key 를 클릭한다

Key 부분에 복사한 public key 를 등록하면 SSH key 등록이 된다

이제 내 Repositories 에 있는 Repository 는 전부 연결이 정상작동 할거다

Front 개발 블로그 (5)

저번에 프로젝트 개발 기본설정을 끝 마쳤으니 이번에 어떻게 프로젝트를 만들지 기획을 해보겠다.
평소라면 종이에 적어서 하겠지만 포스팅을 해야하므로 아이패드를 사용해서 기획을 하도록 하겠다
일단 이 프로젝트의 핵심기능인 여러 게임 플랫폼의 게임 목록을 한곳에서 볼수 있도록 하는 기능이 개발 가능 한지 부터 확인을 해보도록 하겠다
일단 1 차 목표로 steam 과 epic games 의 게임 목록을 불러올수 있는지부터 확인해보겠다
open api 를 확인할때 access token 을 발급 받는 방법, 필요한 api 와 scope 2가지를 유의해서 보면 된다
steam open api :
https://partner.steamgames.com/doc/webapi_overview
https://developer.valvesoftware.com/wiki/SteamWeb_API#GetGlobalAchievementPercentagesForApp.28v0001.29

일단 GetOwnedGames 라는 api 가 있다
내 소유의 게임들 이라는 이름이니 아마 내가 소유하고있는 게임의 목록을 가져올수 있을것으로 보인다

web api 사용법이 적혀있는 부분이다
아직 backend 를 개발하지 않을거기 때문에 일단은 기능 개발이 가능한것만 확인했으니 이제 epic games 를 확인해보도록 하겠다
epic games open api :
https://dev.epicgames.com/docs/services
epic games 는 약 1시간 동안 찾아 보았지만 게임 목록 불러오는 기능을 찾지 못하여 나중에 crawling 을 하든 하도록 하겠다 backend 를 개발할때 헤당 기능 다시 찾아보기로 하고 일단은 넘어가겠다
이제 웹 뷰 기획을 시작하겠다

기획안 1

nav를 좌측에 위치시키고 오른쪽에 Account 관련 정보를 둔다Account 를 클릭하면 팝업이 나타나고 현재 연동중인 플랫폼 (steam, epic games. etc…) 의 현재 상태가 나타나게 한다
이런식으로 진행했을경우 장점은 메뉴, 서브메뉴가 많아져도 편리하게 이용할수 있다

기획안 2

nav 를 header 아래에 위치시키고 기획안 1과 동일하게 Account 를 위치시킨다
이 방법의 장점은 width 를 크게 사용할수 있다
하지만 menu, submenu 가 많아지면 화면이 쉽게 지저분해질수 있으므로 주의를 가해야 한다
일단 당장 필요한 기능을 생각해 보면

  1. dashBoard
    게임 플레이시간 관련 통계, 소유하고있는 게임의 종류 통계,
  2. gameList
    게임 목록 listview,cardview 2가지, 게임 실행
  3. setting
    noti 관련 설정 (평소 게임을 자주 플레이 하는시간에 맞춰서 알림)
    정도가 있겠다.

dashBoard


DashBoard 페이지 기획안이다
사실 차트는 어떤 데이터를 어떻게 보여줄지 생각하고 만들어야 해서
어떤 데이터를 보여줄지 생각을 하면서 기획을 했지만 아직 어떤데이터를 가져올수있는지 확실하지 않아서 개발하면서 바뀔거같다

게임 목록페이지를 card view 와 list view 2벌을 만들 예정이다
일단 기본적인 목록만 추가했고 filter 와 order by 를 추가할 예정이다
마지막으로 setting 을 추가해야하는데 setting 페이지는 가장 마지막에 개발할 예정으로 추후에 다시 다루도록 하겠다.

Front 개발 블로그 (1)

이번에 석가탄신일,근로자의 날,어린이날 등 긴 연휴가 생겨서 뭐라도 해봐야할거같아서 블로그 포스팅을 하기로 결정했다.
어떤 토이 프로젝트 를 진행해야 재미있고 또 유익하게 코딩을할수 있을까
고민중인지라 내가 부족하다 싶은부분, 또 하고 싶은 기술을 먼저 정리해볼까 한다

Chart

말할것도없이 나는 차트를 못만든다….
회사를 다니면서 실무로 차트를 만들일도 크게 없어서
원하는 차트를 마음대로 그리지 못한다
그래서 이번 프로젝트에서 복잡한 차트를 만드는것이 목표이다
추가로 평소 사용하는 chart.js 가 아닌 D3 를 사용하는것이 목표이다
(어디까지나 목표 설정이다..)

Socket + Web notification

실시간으로 서버와 통신을 해보고싶다.
소켓을 이용해서 실시간으로 내가 만들 웹 페이지를 보고 있지않아도
노티를 띄우는게 목표이다 .
평소 해보고 싶었던 것 중 하나인데 프론트 작업 뿐만 아니라 백엔드 자원도 필요하기때문에 쉽게 만들기 힘들어서 도전해 보지 못하였다
그래서 이번 프로젝트에서는 소켓, 웹노티, ServiceWorker 까지 사용해서 기능을 하나 만들어 보고 싶다

Store

현재 내가 회사에서 진행하고있는 프로젝트는 Store 가 크게 필요없다
그래서 인지 항상 스토어를 사용할지 말지 고민을 하고 무턱대고 적용하고 필요없는거같아서 제거하고 를 반복한다
사실 이번에 내가 만들 프로젝트가 Store 가 필요할거 같지는 않지만
평소 Store 를 자주 사용하지 않는탓에 이에 대한 이해도가 부족한거같으니 생산성이 떨어지더라도 한번 해보려고 한다

TypeScript

공부하기 가장 싫어하는 기술이다 타입을 적었을 때 컴포넌트를 재사용 할 경우 편리하긴하다
하지만 작성하기 너무 귀찮아서 잘 사용하지 않는다 …
사실 좋은 습관도 아니고 이번기회에 타입스크립트를 strict 하게 적용하며 타입스크립트에 적응하고싶다
개발시간이 중요한 프로젝트가 아니니 천천히 확실하게 적용할 생각이다
이정도 기술이 들어간다 가정하고 토이프로젝트를 진행하려 한다
그래서 어떤 토이 프로젝트를 진행하면 좋을까

gog galaxy

gog 갤럭시라는 앱이 있는데 이거랑 비슷한 프로젝트를 진행하는것도 괜찮을거같다.
스팀, 에픽게임즈 에 여러 게임이 있어서 가끔 같은 게임을 세일 한다고 덜컥 사버리는 경우도 있다
이를 방지하기 위해 gogGalaxy 앱을 사용하고있는데 불편한 점이 몇가지 있어서 이를 개선하고 내가 편하게 사용할수있게 만드는것도 재미있지 않을까 싶다.

숙박 예약 웹

야놀자, 호텔스 컴바인 등 여러 앱들이 이미 존재하고있다.
그래서 여러 웹사이트의 숙박정보들을 crawler 를 통해 정보를 수집한뒤 가격비교 를 하는 프로젝트도 나름 재미있을거같다
여러 숙박업체의 지도상의 위치는 동일할태니 위치로 묶어서 가격비교를 하면 될거같다
이외에 더 재미있어보이는 프로젝트 가 있으면 고려해보겠지만
딱히 생각이 나오질 않는 관계로 여기 까지로 하고 다음편에는 기본적인 개발 설정을 해보려고한다.
