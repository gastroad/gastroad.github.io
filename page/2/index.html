npm 을 이용하여 필요한 라이브러리 다운로드

일단 프로젝트 디랙토리 부터 생성을 하자

백엔드 부분도 개발을해서 git hub 에 올릴 예정이기 때문에
폴더만 미리 먼저 만들어 두었다
그후 터미널로 가서 git init 을 해주면

위 사진 처럼 저장소를 초기화 했다고 나온다

그후 ls -al 명령어를 통해 git 이 정상적으로 설치가 된것을 확인할수 있다
이제 savemoney-front 로 이동해서 npm init 을 친후 엔터를 누르다 보면

위 사진 처럼 npm init 설정이 될것이다
이후 savemoney-front 디렉토리를 확인해보면

pakage.json 이 생성된것을 확인할수 있다.
이제 기본적으로 필요한 라이브러리 를 설치할려고 한다

기본적으로 react,react-dom 과 webpack 을 dev에 설치 했다
npm install 과 npm install — save-dev 의 차이점은

위 사진 처럼 dev dependencies에 설치할것인지 dependencies 에 설치할지 정할수 있다
dev dependencies 에 설치된 라이브러리 들은 prd 로 빌드했을때 추가 되지않는다 즉 개발에 필요한 도구들은 dev dependencies 에 prd, 개발 둘다 필요한 경우에는 dependencies 에 추가 하면 된다
npm install 이 정상적으로 작동을했으면 node_modules 라는 폴더가 생겼을것이다 이거까지 git 에 올리면 파일 용량이 엄청 커지므로 git 에 올릴때는 헤당하는 폴더는 무시하게 .gitignore 를 작성해 주는것이 좋다

다시 루트 디렉토리로 돌아와 .gitignore 를 작성해준다
node_modules 와 나중에 만들 build 폴더까지 미리 ignore 에 추가해두면 git 에 push 를 해도 node_modules 와 build 폴더는 무시된다

이것저것 개발에 필요한 라이브러리를 설치하고 나면 이런 모습이다

Front 개발 블로그 (1)

이번에 석가탄신일,근로자의 날,어린이날 등 긴 연휴가 생겨서 뭐라도 해봐야할거같아서 블로그 포스팅을 하기로 결정했다.
어떤 토이 프로젝트 를 진행해야 재미있고 또 유익하게 코딩을할수 있을까
고민중인지라 내가 부족하다 싶은부분, 또 하고 싶은 기술을 먼저 정리해볼까 한다

Chart

말할것도없이 나는 차트를 못만든다….
회사를 다니면서 실무로 차트를 만들일도 크게 없어서
원하는 차트를 마음대로 그리지 못한다
그래서 이번 프로젝트에서 복잡한 차트를 만드는것이 목표이다
추가로 평소 사용하는 chart.js 가 아닌 D3 를 사용하는것이 목표이다
(어디까지나 목표 설정이다..)

Socket + Web notification

실시간으로 서버와 통신을 해보고싶다.
소켓을 이용해서 실시간으로 내가 만들 웹 페이지를 보고 있지않아도
노티를 띄우는게 목표이다 .
평소 해보고 싶었던 것 중 하나인데 프론트 작업 뿐만 아니라 백엔드 자원도 필요하기때문에 쉽게 만들기 힘들어서 도전해 보지 못하였다
그래서 이번 프로젝트에서는 소켓, 웹노티, ServiceWorker 까지 사용해서 기능을 하나 만들어 보고 싶다

Store

현재 내가 회사에서 진행하고있는 프로젝트는 Store 가 크게 필요없다
그래서 인지 항상 스토어를 사용할지 말지 고민을 하고 무턱대고 적용하고 필요없는거같아서 제거하고 를 반복한다
사실 이번에 내가 만들 프로젝트가 Store 가 필요할거 같지는 않지만
평소 Store 를 자주 사용하지 않는탓에 이에 대한 이해도가 부족한거같으니 생산성이 떨어지더라도 한번 해보려고 한다

TypeScript

공부하기 가장 싫어하는 기술이다 타입을 적었을 때 컴포넌트를 재사용 할 경우 편리하긴하다
하지만 작성하기 너무 귀찮아서 잘 사용하지 않는다 …
사실 좋은 습관도 아니고 이번기회에 타입스크립트를 strict 하게 적용하며 타입스크립트에 적응하고싶다
개발시간이 중요한 프로젝트가 아니니 천천히 확실하게 적용할 생각이다
이정도 기술이 들어간다 가정하고 토이프로젝트를 진행하려 한다
그래서 어떤 토이 프로젝트를 진행하면 좋을까

gog galaxy

gog 갤럭시라는 앱이 있는데 이거랑 비슷한 프로젝트를 진행하는것도 괜찮을거같다.
스팀, 에픽게임즈 에 여러 게임이 있어서 가끔 같은 게임을 세일 한다고 덜컥 사버리는 경우도 있다
이를 방지하기 위해 gogGalaxy 앱을 사용하고있는데 불편한 점이 몇가지 있어서 이를 개선하고 내가 편하게 사용할수있게 만드는것도 재미있지 않을까 싶다.

숙박 예약 웹

야놀자, 호텔스 컴바인 등 여러 앱들이 이미 존재하고있다.
그래서 여러 웹사이트의 숙박정보들을 crawler 를 통해 정보를 수집한뒤 가격비교 를 하는 프로젝트도 나름 재미있을거같다
여러 숙박업체의 지도상의 위치는 동일할태니 위치로 묶어서 가격비교를 하면 될거같다
이외에 더 재미있어보이는 프로젝트 가 있으면 고려해보겠지만
딱히 생각이 나오질 않는 관계로 여기 까지로 하고 다음편에는 기본적인 개발 설정을 해보려고한다.
