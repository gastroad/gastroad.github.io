react-query를 사용해보자(1)

react-query 의 사용은 react-query를 사용해보자(2)에서 부터 사용을 시작합니다. react-query 사용이 궁금하신 분들은 바로 해당 포스트를 보러가주시면 감사하겠습니다.

react-query 를 사용해보려고 한다.

redux 를 이용하다 보면 비동기 요청마다 미들웨어를 추가해주는 작업은 상당히 번거롭게 느껴진다.
간단한 요청들은 react-query를 사용해보려 한다.

react-query 에 대해 알아보고 react-query 가 아닌 swr, rtk-query 랑도 비교 해보고 사용해보려고 한다.

먼저 react-query 의 동기부터 살펴보자

Caching… (possibly the hardest thing to do in programming)
Deduping multiple requests for the same data into a single request
Updating “out of date” data in the background
Knowing when data is “out of date”
Reflecting updates to data as quickly as possible
Performance optimizations like pagination and lazy loading data
Managing memory and garbage collection of server state
Memoizing query results with structural sharing

캐시,
동일한 데이터에 대한 여러 요청을 단일 요청으로 중복 제거,
백그라운드에서 오래된 데이터 알고 업데이트
데이터 업데이트를 최대한 신속하게 반영
페이지로딩 및 lazy 로딩 데이터와 같은 성능 최적화
메모리 관리 와 서버 state 의 garbage collection
구조적 공유와 query 결과 Memoizing

목록만 봐도 아찔해진다 이쯤되면 안쓰면 안될거같은 기분마저 들기 시작한다.

일단 한번 사용해보려고 한다.
이를 위해서 api를 호출할 필요가 있다

다른 프로젝트를 만들땐 아래와 같이 mock api 를 만들어서 사용했었다

하지만 이럴경우 여러 상황을 테스트 해보기 번거로워 그냥 node 로 간단한 웹 서버를 만들어 보려 한다.

우선 터미널에 들어가 간단한 환경 설정을 해주자

node-server 라는 디렉토리를 만들고 express 랑 nodemon 을 설치해 주었다

pakage.json 에 nodemon 을 통해서 app.js 를 실행하는 scripts 를 추가해 주었다.

이제 텍스트 에디터를 통해서 간단한 웹서버를 만들어보려한다

이제 http://localhost:3000/api/portfolio 로 접속해보면 json 이 잘 내려오는것을 확인할수 있다.

이제 api 를 생성했으니 react에서 호출을 해보려 한다.

포트폴리오 목록 업데이트 하기 귀찮아서 만든 레포지토리가 있는데
이 레포지토리를 이용해서 만들어 보겠다.

현재 포토폴리오 목록은 GitHub Pages 를 통해 배포가 되어있는 상태이다
서버를 통한 api 호출을 하도록 수정하게 되면
웹서버를 배포해 주어야 정상작동한다.
이렇게 되면 비용이 발생해버리니… 그냥 해당 레포지토리를 복사해서 설정을 해보려고 한다.

혹시나 따라해보고싶으신 분이 계시다면 깃허브링크 에서 head 를 610b4f0 에 위치시키고 따라해주시면 감사하겠습니다.

우선 api 호출해 보기 위해 webpack 설정을 수정해 주어야 한다

localhost:3000/api 로 이동된다.
