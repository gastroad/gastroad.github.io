hosts

오늘은 hosts 파일 수정하는 법에 대해서 알아보도록 하겠다

host 파일 이란

hosts 파일은 운영 체제가 호스트 이름을 IP 주소에 매핑할 때 사용하는 컴퓨터 파일이다.

쉽게 브라우저에 도메인 을 입력했을때 해당하는 IP 주소에 매핑을해줘 찾아갈수 있게 하는 파일이다.

가장 대표적으로 localhost 가 있다
127.0.0.1 IP 를 localhost 라는 도메인에 매핑을 해 브라우저에 localhost 라 입력을해도 알아서 127.0.0.1 로 찾아 가게 된다.

자 이제 그러면 언제 어떻게 사용하는가

프론트엔드 개발을 하는 경우 주로 로컬 개발 서버를 띄워서 작업을 진행 할 것이다.
그러면 보통 localhost 즉 127.0.0.1 에 웹서버를 띄워서 작업을 진행 할 탠데
카카오톡이나 페이스북 등 소셜 로그인(oauth) 기능을 구현해야하는 상황에서 hosts 파일을 수정하면 편리하게 개발할수 있다.

카카오톡을 예시로 들겠다
소셜 로그인을 구현하는 방법까지는 다루지 않도록 하겠다.

카카오 로그인 기능을 구현할때 크게 3가지 단계로 구분해서 개발을 진행했다

  1. code 발급받기
  2. 발급받은 code 를 가지고 access token 발급받기
  3. 발급받은 access token 을 가지고 auth token(login)발급받기

이중 첫번째 code 발급 받는 과정에서 hosts 파일을 수정하여 유용하게 사용할수 있다
code 를 발급 받으려면 url 을 redirect 시켜야 한다.
이 과정에서 카카오톡 어플리케이션에 Redirect URI 를 미리 등록해 두어야 한다.

이때 Redirect URI 에 localhost를 넣을 순 있지만.
서비스를 계속 운영하면서 localhost를 넣어둔채로 작업하기에는 다소 찜찜하기에
본인의경우 production, stage 에서 redirect 를 받을 URI 만 넣어주었다.

이 경우 oauth 를 개발하면서 계속 확인해보고 테스트 해봐야하는데 그때마다 배포해서 확인해 볼수는 없으니
이때 hosts 파일을 수정해서 작업이 가능하다.

자 그러면 이제 실제로 hosts 파일을 수정하는 방법에 대해 알아보도록하자

hosts 수정은 정말 간단하다 우선 하단에 링크해둔 위키백과에서 본인에 맞는 os 의 hosts 파일 위치를 찾아보자.
맥 OS X 10.2 이상 을 사용하고있으니 /etc/hosts 에 있을것이다 그럼 바로 터미널로 들어가서 파일을 수정해보자

여기서 왼쪽에 IP 오른쪽에 도메인을 입력 하면 해당하는 도메인에 해당하는 IP 로 매핑을 해준다

위와 같이 설정해두고 naver.com 에 접속하면 127.0.0.1 즉 localhost 가 보이게 된다.
만약 위와같이 설정을 했는데도 적용이 잘 안된거같다 싶으면 사용하고있는 브라우저를 완전히 껐다가 다시 켜보자.

이렇게 Hosts 를 수정해서 ouath 를 보다 편하게 개발할수 있다.

물론 ouath 개발을 편하게 하는 방법중에는
환경변수 같은걸 이용해서 local stage production 다 다른 카카오톡 어플리케이션을 바라보게 해서
localhost 를 redirect 하여 작업을 해도 무방하다

본인이 개발할수 있는 상황에 맞는 편한 방법을 사용해서 개발하면 된다고 생각한다.
