react-query를 사용해보자(2)

어제 간단한 웹서버를 만들고 api 동작 까지 확인했으니 본격 적으로 react-query 를 사용해보려 한다.

우선 당연하게도 react-query 를 설치해주자

바로 사용을 시작해보자
우선 react-query 를 이용하기위해서는 quertclient 를 provider 에 넣어 줘야 한다

이제 api 를 호출하는 함수를 만들어 두자

api 를 너무 과하게 만든거같다… 기왕 과하게 만들거면 좀 이쁘게 만들껄 그랬다

우선 바로한번 호출 해 보자

우선 useQuery 를 통해 api 호출이 가능하다.

useQuery에 우선 필수 값인 queryKey 와 queryFn 만 넣어서 호출을 확인해 보았다.

이제 차근차근 useQuery에 어떤 옵션들이 들어가고 각각 어떤일을 하는 친구인지 알아보자

우선 queryKey 부터 살펴 보자

그러면 enabled 도 궁금해 진다

enabled 는 api 를 호출한 후 그 데이터를 가지고 추가 호출이 일어나야하는 상황에서 사용할수 있을거 같다.

그러면 queryFn 은 뭐하는 친구일까

옵션은 여기까지만 확인해보고 return 항목에 대해서 알아보겠다
우선 지금 당장 필요한건 요청의 상태와 에러 그리고 결과가 필요하다

잘 작동하는것을 확인했다.

자세한건 tanstack.com 에서 확인해 볼수 있다 useQuery 에 정말 수많은 옵션과 return 이 있다.

이제 get api는 확인을 했으니
post 와 이에 따른 update 를 해보려고한다.

useMutation 이라는 훅을 사용할 예정이다.

useMutation 훅에 mutationFn 에 post 할 api 를 넣어 주었다.
이후 onSuccess 옵션을 통해 api 가 성공했을때 portfolioList 를 다시 받아오는 작업을 진행하였다. onError 같이 실패했을경우 에러처리도 가능하니 뭔가 필요한 상황이 오면 document 를 다시 봐야 겠다.

여기까지 해보고 나니 굳이 다른 api 를 만들어 볼 필요가 없었던거같다.

기존에 rtk 를 사용해서 createAsyncThunk 를 사용해서 extraReducers 를 추가해주는 방식보다 훨씬 간편하게 사용할수 있는것 같다.

내일은 rtk-query 를 사용해 보고 react-query 랑 비교 해보려고 한다.

참고: https://tanstack.com/query/latest/docs/react/overview
