docker kubernetes 이용하여 aws 배포하기 2

우선 Docker 부터 설정을 해보려 한다
결국 kubernetes,ligthsail 둘다 컨테이너화 된 docker 이미지가 있어야 사용할수 있으니 Docker 사용법은 조금 자세하게 알아보겠다.

Docker

오늘 해볼 일은 Docker 를 사용해서 내가 예전에 만들어둔 레포를 image 화 해보는것이다.

우선 Docker 를 설치해보자
Docker 설치의 경우 Docker 에서 설치할수 있다 현재 m2 맥북을 사용하고있으므로 인텔 칩 전용 버전을 다운로드 했다

Docker 를 설치하고 실행한다면 아래와 같은 화면이 나올것이다

이제 docker 설치는 확인했으니 Dockerfile 을 생성해보자

vscode 에서 Dockerfile 을 생성하니 extension 도 설치를 추천하길래 설치해줬다

이제 DockerFile 을 작성하겠다

DockerFile 을 작성했으면 빌드가 잘 되는지 확인해보자

잘 작동하는지는 모르겠지만 일단 빌드에는 성공을 했다
이제 이미지를 빌드했으니 앱 컨테이너를 시작해보자

이제 아래 명령어를 통해 정상적으로 작동하고있는지 확인 해보자

아무것도 나오지 않는다

Docker Desktop 에서 컨테이너의 로그를 보니 아래 사진같이 나왔다

package.json 에 프로젝트를 로컬에서 구동하는 모든 코드들을 devDependencies 에 작성해서
production 으로 install 하면 안되는것이 당연하다

우선 구동 확인을 위해서 production 이 아닌 그냥
npm install 으로 작성해 보니 잘 작동했다.

하지만 서비스 배포 목적으로 도커를 사용할예정이니 작업물을
production 으로 빌드해서 컨테이너를 띄워보겠다.

참고로 pakage.json 의 scripts 는 다음과 같다

이제 다시 시도해보니

빌드에 실패했다
일단 에러를 보니 권한 관련된 문제인거같아 sudo 로 일단 실행해보자

error getting credentials 뭐라 하는것을 보니 계정과 관련된 문제가 생긴것으로 추정된다.

도커 로그인을 진행하고 다시 시도해도 같은 문제가 발생한다.

우선 가동을 우선으로 Dockerfile 을 다시 작성해보았다

nginx 이미지를 가져오는데에서 문제가 생겼으니 그냥 최신버전을 가져오게 수정하였다
수정하는김에 node 도 그냥 최신버전을 가져오게 수정하였다.

이제 다시 시도해 보니 성공적으로 빌드가 되었다!!

이제 컨테이너에 다시 올리고
마지막으로 nginx 관련 설정과 명령어를 Dockerfile에 추가하고
다시 빌드해서 컨테이너에 올려보겠다.

이제 localhost 에 접속하면 정상 작동한다.

이번 포스팅을 하면서 웹서버 와 배포 환경에 얼마나 지식이 없는지 뼈아프게 실감했다.
포스트에 다쓰기엔 너무 길어질거같아 생략한 nginx 설정과 빌드 cahe 관련된 문제도 있었다. 다음에 여유가 있을때 nginx 관련내용도 포스팅해보도록 하겠다 우선 docker 컨테이너를 빌드하는데 성공하였으니
다음에는 aws kubernetes 에 대해 알아보도록 하겠다.
