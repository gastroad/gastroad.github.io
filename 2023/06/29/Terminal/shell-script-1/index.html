컴포넌트 생성이 번거로우신가요?

최근 atomic 패턴으로 개발을 진행하다 보니 컴포넌트를 생성하는 것에 대한 번거로움이 생겼다.
그래서 컴포넌트 생성을 조금이나마 편하게 하고자 shell script를 짜 보았다.

우선 컴포넌트를 생성할 때마다 하는 작업을 확인해 보자

  1. component 디렉토리 생성
  2. component 파일 생성
  3. index 파일 생성
  4. test 파일 생성
  5. story 파일 생성
  6. style 파일 생성

각 파일들을 생성하고 기본적으로 작성해야 하는 내용을 확인해 보자

component의 경우 위와 같은 형태를 기본으로 개발을 진행한다. 기호의 따라 function으로 생성해도 무방하다

index 파일의 경우 컴포넌트 접근을 위해 작성해 준다
index 가 아닌 package.json 을 만들어 주어도 무방하다

story 파일의 경우 대부분 비슷한 형태를 가지고 있다.
대부분 여기서 args를 수정해 주거나
decorators를 추가해 주는 정도다.

style, test의 경우 파일만 생성해 두고 내용은 컴포넌트에 따라 완전히 바뀔 수 있으므로 그냥 빈칸으로 두겠다.

위 내용을 바탕으로 shell-script를 작성하면 다음과 같이 나온다

간단하게 사용된 문법을 설명하자면

componentName 을 입력받는다

생성할 파일 배열을 만들어준다

files의 개수만큼 for 문을 돈다

files 배열에 있는 파일을 생성해 준다

file 이 “” 와 일치할 경우 ;; 전까지의 명령어를 실행한다

cat 명령어를 통하여 파일 내용을 수정해 준다.

터미널에서 아래 명령어를 입력하면 잘 작동하는 것을 확인할 수 있다.

다만 이렇게 될 경우 디렉토리가 생성된 이후 직접 원하는 위치에 배치해야 하는 번거로움이 있다.

그래서 어떤 위치에 배치시킬지 입력을 받도록 하자

select를 이용해 어떤 컴포넌트를 생성할지 입력을 받아 componentPath를 변경하게 만들었다.
