컴포넌트 생성이 번거로우신가요? 2

지난번에 shell-script를 통하여 컴포넌트 생성 자동화를 진행하였다.
사용하다 보니 만들어진 코드를 크게 수정하진 않지만 경로를 수정해 주고 파일을 계속 복사 붙여넣기 해야 해 조금 번거로운 감이 있다.

어떻게 고도화를 하면 좋을까 고민하다
vscode extension으로 만들면 편리하지 않을까라는 생각이 들어
이번에 vscode extension으로 만들어 보려 한다.

우선 vscode extenstion 을 어떻게 만드는지 확인해 보았다.
친절하게도 vscode docs 에 해당 문서가 있어 우선 따라 해 보았다.

우선 node, git 이 설치되어 있는지 확인을 해 보았다.

당연하겠지만 잘 설치되어 있다
그다음 yeoman, generator-code 를 설치하고 실행해 주었다


typescript로 extension 을 만들어준다고 선택을 하면 프로젝트가 자동으로 생성된다.

생성된 프로젝트를 살펴보자

실제 생성해 보면 주석으로 각 코드에 대한 설명이 있다
간단하게 요약을 하면
registerCommand의 첫 번째 인자가 pacakge.json에 command에 있어야 한다.

그리고 commands에 입력된 command는 extension을 실행시켰을 때
command palette에 검색하면 실행시킬 수 있다.

내가 하고 싶은 건 explorer에서 우클릭으로 컴포넌트를 생성하는 extension 을 만들고 싶으니 vscode docs를 살펴보았다.

explorer context menu에서 navigation에 커맨드가 나타나면 될 것 같다.

다음과 같이 작성하고 확인해 보니 이쁘게 잘 생겼다!

이제 본격적인 command를 작성해 보도록 하겠다.
추후에 확장성을 고려해서 extension.ts에는 커맨드 등록만 하려고 한다

각각의 command는 command 디렉토리를 생성해서 선언해 두었다

createComponent 가 실행됐을 때 컴포넌트 이름을 입력받고
입력받은 이름을 바탕으로 디렉토리를 생성한다.
그 후 생성할 파일을 선택한다.
component, index, 선택한 파일을 생선 된 디렉토리 하위에 생성하는 코드를 작성하였다.

이제 잘 작동하는지 테스트해보자


컴포넌트가 이쁘게 잘 생성된것을 확인하였다!

코드 전문

vscode 익스텐션을 처음 만들다 보니 여러 시행착오를 많이 겪었다
디렉토리 구조 및 코드도 조금 지저분한 느낌을 받는다
나중에 여유가 된다면 잘 만든 익스텐션을 한번 참고해 봐야 할 것 같다
그래도 document 가 잘 작성되어 있어서 어찌어찌 완성할 수 있었던 것 같다.

개발할 때 불편한 점이 있다면 계속해서 추가 개발할 예정이다.
