Front 개발 블로그 (3)

저번에 필요한 라이브러리 설치까지 완료했다 글을 쓰면서 같이 작업을 진행하다 보니 생각보다 시간이 오래걸려 오늘 목표는 짧게 잡기로 정했다
오늘의 TODO LIST는

  1. webpack config 하기
  2. 프로젝트 구동 하기

이 두가지를 해보려고 한다

webpack config 하기

일단 package.json 과 같은 위치에 webpack.config.js 파일을 생성한다.
그리고 웹팩이 번들링 할 파일을 생성한다
나는 src/index.tsx 를 생성해 주었다

이제 웹팩이 번들링을 할 index.tsx 파일의 위치를 webpack.config.js 에 적어준다

그 후 웹팩 빌드 결과물이 나올 위치를 지정해 준다

__dirname 은 root 에서 부터 현재 위치를 나타낸다
현재 위치에서 build 폴더에 webpack 이 빌드 하고 나온 결과물 을 저장한다
이제 저번에 미리 package.json 에 작성 해둔 scripts 를 터미널에서 실행해보겠다

위 사진 처럼 나왔으면 성공이다
이제 build 디렉토리에 가서 확인해보면
webpack.config.js 에서 output 에 filename 으로 작성한
my-first-webpack.bundle.js 가 생성되었을것이다

이제 필요한 loader 를 추가 하도록 하자

  1. babel 은 호완성을 위해서 무조건 설치해주는것이 좋다
  2. 개발을 하면서 scss 를 사용할것이기 때문에 style-loader,css-loader 뿐만아니라 sass-loader 를 추가해 주었다
  3. file-lodaer 를 추가해줘야 사진을 올릴수가 있으니 추가해주었다
  4. 폰트 적용을 위해서 추가해 주었다
    웹 페이지 디자인이 아무리 깔끔해도 폰트가 구리면 페이지가 전체적으로 낡은 느낌이나는거 같아서 폰트는 귀찮아도 꼭 추가하는것을 추천한다.

  1. alias 는 @src @components @utils 3개를 작성해 주었다
    프로젝트 내부에서 import 를 할때 @src 를 입력해서 절대경로로 path 를 설정해 줄수 있다
  2. extenstions 는 확장명없는 파일을 허용하지 않는 기능이다
    webpack docs 를 보면 이 기능을 사용하면 ./foo.js 를 ./foo 로 입력할수 있게 해준다
    이제 devServer 설정을 해줄것이다

  1. host 0.0.0.0 은 자기 자신을 가르키는 ip 이다
  2. https 의 기본포트는 443 이므로 번거롭게 port 를 입력해주기 귀찮으면 443 으로 설정하면 된다
  3. disableHostCheck 는 true 로 설정하면 호스트 검사를 무시한다
    dns rebinding attacks 에 취약하다고 webpack docs 에 쓰여져있지만
    혼자서 토이프로젝트를 진행하는데 문제될부분은 없어서 사용했다
  4. historyApiFallback 은 api 를 사용할때 404 응답대신 index.html 페이지를 제공해준다
  5. https 는 https:로 접근할수 있게 해준다
    이제 devServer 설정이 끝났다 마지막으로 plugins 만 추가하면 webpack 설정은 일단 끝난다(추후 개발하면서 여러번 수정이 있을 예정이다.)

  1. CaseSensitivePathsPlugin 은 필요한 모든 모듈의 전체 경로를 디스크상의 실제 경로와 정확히 일치시키는 플러그인이다
    파일 시스템 이 달라 대소문자 때문에 충돌을 일으킬 경우를 줄여준다

  2. HtmlWebpackPlugin 은 html 파일 생성을 단순화하여 웹팩 번들을
    제공한다
    template 에 쓰여있듯이 index.tsx 가 위치한
    폴더에 index.html 도 생성해줘야 한다
    이제 다시 터미널로 돌아와서 npm start 명령어를 입력해 보자

이런식으로 나온다면 성공적으로 웹팩 설정을 끝낸것이다

프로젝트 구동 하기

이제 https://localhost 에 접속하면 아무것도없는 빈 화면이 나올것이다
아무것도 없으면 프로젝트가 정상적으로 빌드가 되고있는건지 안되고있는건지 확인하기 어려우니 index.html 과 index.tsx 를 작성해주도록 하자

index.html 에 body 안에 id=”root” 만 작성했다 이제 저 위치를 잡아서 react의 시작점으로 만들것이다
이제 index.tsx 로 가서

위와 같이 작성한다
root 를 getElementById 해서 app 을 보여주겠다는 뜻이다
그리고 같은 경로에 App.tsx 를

아래와 같이 작성한다
