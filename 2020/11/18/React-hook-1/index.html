React Hook (1)

오늘은 react의 hook 에 대해서 포스팅을 해 볼 생각이다
이미 많은 사람들이 hook 에 대해서 포스팅을 해서 내 포스팅이 영양가가 있을거 같지는 않지만
1년정도 사용하면서 정확하게 모르는 부분도 있기 때문에 정리할겸 글을 써보려 한다

일단 hook 은 2018 ReactConf 에서 발표하고 React 16.8 버전부터 추가된 기능이다
hook의 특징으로는 다음과 같다

  1. 선택적 사용 기존의 코드를 다시 작성할 필요 없이 일부의 컴포넌트들 안에서 Hook을 사용할 수 있습니다.
    그러나 만약 당장 Hook이 필요 없다면, Hook을 사용할 필요는 없습니다.
  2. 100% 이전 버전과의 호환성 Hook은 호환성을 깨뜨리는 변화가 없습니다.
  3. 현재 사용 가능 Hook은 배포 v16.8.0에서 사용할 수 있습니다.

개인적으로 2번 이전 버전과의 호환성 부분이 가장 마음에 들었다 react 에서 class 바로 제거 해주지않아서 hook에 대해서 완벽하게 공부하기전에
리팩토링을 해야하는 불상사가 일어나지않아서 좋았다
버전을 올리지 않는다는 선택지는 없다

hook 을 만든 이유는

  1. 컴포넌트 사이에서 상태와 관련된 로직을 재사용하기 어렵습니다.
  2. 복잡한 컴포넌트들은 이해하기 어렵습니다.
  3. class는 사람과 기계를 혼동시킵니다.

3가지를 말했다

컴포넌트 사이에서 상태와 관련된 로직을 재사용하기 어렵습니다.

React는 컴포넌트에 재사용 가능한 행동을 붙이는 방법을 제공하지않는다
(ex) store 에 component 를 연결하는것)
이걸 해결하기위해 render props 그리고 higher-order components 같은 패턴을 사용하게되는데
이런 패턴을 사용할때 컴포넌트를 재구성해야 하며 코드를 추적하기 어렵게 만든다 한다

hook 을 사용하면 컴포넌트로 부터 상태 관련 로직을 추상화 할수 있고 계층 변화 없이 상태 관련 로직을 재사용할 수 있도록 도와준다고 한다

복잡한 컴포넌트들은 이해하기 어렵습니다.

유지하기 힘든 상태 관련 로직들과 사이드 이펙트가 있는 컴포넌트들을 유지할때가 있다
각 lifecycle method 에 관련없는 로직이 섞여있을때가 있다
이로인해 버그가 쉽게 발생하고 무결성을 쉽게 해친다

hook 을 사용하면 로직에 기반을 둔 작은 함수로 컴포넌트를 나눌수 있다

Clss은 사람과 기계를 혼동시킵니다.

class 는 코드의 재사용성과 코드 구성을 좀 더 어렵게 만들 뿐만 아니라 React를 배우는데 큰 진입장벽이다
javascript 에서 어떻게 this 가 작동하는지 알아야만 했고 대부분의 다른 언어와 는 다르게 작동한다
처음 React 를 공부할때 constructor 라던가 binding 이라던가 사실 이해를 하지못하고 대충 따라서 쳤다 그래서 js를 처음 공부하고 React 를 사용하는 유저의 진입장벽을 크게 낮출수 있을거같다.

hook 을 사용하면 Class 없이 React 기능들을 사용하는 방법을 알려준다

여기까지 hook의 개요에 대해서 알아보았다.
이렇게 기존 React의 많은 단점을 보완한것으로 보이는데 실제로 hook 을 사용했을때 어떻게 변하는지 다음 포스팅에서 알아보겠다

참고 : https://reactjs.org/docs/hooks-intro.html
